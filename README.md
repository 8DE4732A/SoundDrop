# SoundDrop 🔊

通过声音传输文件的 Web 应用，使用 ggwave.js 实现面对面的文件传输。

## 功能特性

- 📁 支持任意类型文件传输
- 📏 最大支持 100KB 文件
- 🔊 通过扬声器播放声音传输
- 🎤 通过麦克风接收数据
- 📊 实时传输进度显示
- 💾 接收完成后可直接下载
- 📱 响应式设计，支持移动端

## 使用方法

### 发送文件

1. 打开网页后选择「发送文件」标签页
2. 点击文件选择区域，选择要发送的文件（最大 100KB）
3. 查看文件信息确认无误
4. 点击「开始发送」按钮
5. 等待传输完成（会听到一系列声音）

### 接收文件

1. 切换到「接收文件」标签页
2. 点击「开始监听」按钮（需要授权麦克风权限）
3. 等待对方开始发送
4. 传输完成后点击「下载文件」保存

## 技术实现

- **前端框架**: 纯原生 JavaScript
- **音频库**: [ggwave.js](https://github.com/ggerganov/ggwave) v0.4.0
- **音频 API**: Web Audio API
- **传输协议**:
  - 文件被分割成 64 字节的块
  - 使用 Base64 编码
  - 先发送文件元信息（文件名、大小、类型）
  - 然后分块发送文件内容

## 使用建议

- 📍 两台设备尽量靠近（1-2米内效果最佳）
- 🔇 保持环境相对安静
- 🔊 发送设备音量调到中等偏上
- ⏱️ 传输速度较慢，100KB 文件约需 2-3 分钟
- 🔄 如果传输失败，可以刷新页面重试

## 本地运行

由于使用了麦克风和扬声器 API，需要通过 HTTP 服务器运行：

```bash
# 使用 Python
python -m http.server 8000

# 或使用 Node.js
npx http-server
```

然后访问 `http://localhost:8000`

## 浏览器兼容性

- Chrome / Edge (推荐)
- Firefox
- Safari

需要支持以下 API：
- Web Audio API
- MediaDevices API
- FileReader API

## 注意事项

- ⚠️ 仅适用于小文件的面对面传输
- ⚠️ 传输速度较慢，适合演示和特殊场景
- ⚠️ 需要在安静环境中使用
- ⚠️ 建议使用 HTTPS 或 localhost（麦克风权限要求）
